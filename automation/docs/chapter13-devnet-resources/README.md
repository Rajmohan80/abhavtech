# Chapter 13: Cisco DevNet Resources Reference

Comprehensive reference guide for Cisco DevNet resources including Terraform providers, Ansible collections, DevNet Sandboxes for testing, API documentation, and learning labs.

## What You'll Learn

### Terraform Providers
Official Terraform providers for Cisco platforms:

- **CML2**: `ciscodevnet/cml2` - Modeling Labs automation
- **DNAC**: `cisco-en-programmability/dnacenter` - DNA Center provisioning
- **ISE**: `CiscoISE/ise` - Identity Services Engine
- **SD-WAN**: `ciscodevnet/sdwan` - vManage templates
- **Webex**: `CiscoDevNet/webex` - Webex Calling/Contact Center

### Ansible Collections
Galaxy collections for Cisco automation:

- `cisco.ios` - IOS/IOS-XE configuration
- `cisco.nxos` - Nexus platform management
- `cisco.dnac` - DNA Center API modules
- `cisco.ise` - ISE policy management
- `cisco.meraki` - Meraki dashboard automation

### DevNet Sandboxes
Free lab environments for testing:

- DNA Center Always-On Sandbox
- ISE 3.x Sandbox
- SD-WAN 20.x Sandbox
- Webex Contact Center Sandbox
- CML Personal instances

### API Documentation
Official API references:

- DNA Center API (developer.cisco.com/docs/dna-center)
- ISE ERS/pxGrid API
- vManage REST API
- Webex Calling/Teams/Contact Center APIs
- FMC REST API

### DevNet Learning Labs
Guided tutorials and workshops:

- Terraform for Network Engineers
- Ansible Network Automation
- DNA Center Programmability
- SD-WAN API Automation
- ISE pxGrid Integration

## DevNet Community

!!! tip "DevNet Support"
    DevNet Community forums provide peer support for automation questions. Cisco TAC does not support custom automation code, but DevNet forums connect you with engineers solving similar problems.

!!! note "Sandbox Limitations"
    DevNet Sandboxes are shared environments with limited write access. Use CML Personal for full control and persistent configurations.
