# Implementation Roadmap


## Phase 6: GCP Vertex AI + Azure SASE (16 Weeks)


**Week 1-2: Planning & Design**
- [ ] GCP project setup (abhavtech-wxcc-prod)
- [ ] Azure subscription setup (ExpressRoute, Virtual WAN)
- [ ] Order ExpressRoute circuits (3× 10G, 12-week lead time)
- [ ] Order Azure Virtual WAN hub provisioning
- [ ] Design review with Cisco TAC (SD-WAN integration)

**Week 3-4: GCP Foundation**
- [ ] Create GCP VPCs, subnets (not needed for WxCC, but for future)
- [ ] Configure VPC Service Controls perimeter
- [ ] Set up Private Service Connect endpoints
- [ ] Configure Cloud DLP API
- [ ] Enable Vertex AI APIs

**Week 5-8: GCP Vertex AI Implementation**
- [ ] Deploy CCAI Platform (Contact Center AI)
- [ ] Configure Agent Assist
- [ ] Set up Dialogflow CX (IVR flows)
- [ ] Create BigQuery dataset (wxcc_analytics)
- [ ] Configure Cloud Storage bucket (call-recordings)
- [ ] Deploy Vertex AI Feature Store
- [ ] Enable Model Monitoring
- [ ] Integrate WxCC → GCP (webhooks, API exports)

**Week 9-10: Azure ExpressRoute (Once circuits delivered)**
- [ ] Configure ExpressRoute circuits with carriers (Tata/BT/AT&T)
- [ ] Set up BGP peering on SD-WAN edges (C8500-12X)
- [ ] Verify Office 365 route propagation
- [ ] Test latency (target <10ms)
- [ ] Migrate 10% of users to ExpressRoute (pilot)

**Week 11-12: Azure Virtual WAN**
- [ ] Provision Virtual WAN hub (Central India)
- [ ] Deploy VPN gateway (VpnGw3)
- [ ] Configure IPsec tunnels from 15 branches
- [ ] Set up BGP peering
- [ ] Deploy Azure Firewall (integrated with Virtual WAN)
- [ ] Test connectivity (branch → Azure SQL via private endpoint)

**Week 13-14: Azure Private Link**
- [ ] Create private endpoints (5 services: SQL, Storage, Key Vault, ACR, etc.)
- [ ] Configure Private DNS zones
- [ ] Update on-prem DNS forwarding to Azure DNS
- [ ] Disable public IPs for all PaaS services
- [ ] Test connectivity from branches

**Week 15: SD-WAN Cloud OnRamp Configuration**
- [ ] Configure vManage Cloud OnRamp policies
- [ ] Set up application-aware routing (Office 365 → ExpressRoute, Azure → Virtual WAN, Internet → Umbrella)
- [ ] Define SLA thresholds and failover policies
- [ ] Test failover scenarios (ExpressRoute down → DIA)

**Week 16: Production Cutover & Validation**
- [ ] Migrate remaining users to ExpressRoute (100%)
- [ ] Enable WxCC → GCP Vertex AI integration (100% of calls)
- [ ] Validate end-to-end flows (user → Azure → GCP)
- [ ] Performance testing (latency, throughput, MOS)
- [ ] Documentation handover
- [ ] Training for NOC team
