## Traffic Flow Decision Matrix


| Source | Destination | Path | Inspection | Latency Target |
|--------|-------------|------|------------|----------------|
| **Branch User** | **Corporate App (10.252.x.x)** | MPLS → On-Prem DC | FTD Firewall | <30ms |
| **Branch User** | **Office 365 (52.x.x.x)** | ExpressRoute → Microsoft | None (trusted) | <10ms |
| **Branch User** | **Azure SQL (10.100.x.x)** | IPsec → Azure Virtual WAN → Private Endpoint | Azure Firewall | <20ms |
| **Branch User** | **Salesforce / Internet** | DIA → Umbrella SASE → Internet | Umbrella (DNS, Web, FW) | <80ms |
| **Admin (Mumbai)** | **GCP Console** | DIA → Umbrella SASE → Internet → GCP | Umbrella + TLS 1.3 | <50ms |
| **WxCC Cloud** | **GCP Vertex AI** | Cloud-to-Cloud (Cisco backbone) | VPC Service Controls | <5ms |
| **GCP Vertex AI** | **WxCC API** | Cloud-to-Cloud | VPC Service Controls egress | <5ms |
| **Azure App** | **GCP API** | N/A (no direct connectivity) | Via internet if needed | N/A |
